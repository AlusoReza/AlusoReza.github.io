## Session 86: Fix sidebar flash when crossing 1235px breakpoint

### Prompt
Sidebar flashes on top of all content for an instant when resizing from desktop to mobile (and occasionally mobile to desktop). The sidebar panel appears fully visible, overlapping content, then disappears.

### Root cause
Media query sets `transition: transform 0.3s ease` on `.sidebar`. When the media query activates, the browser sees `transform: none → translateX(-100%)` and starts a 0.3s transition. During this transition the sidebar has `opacity: 1` (from the media query) and is partially on-screen (transition not yet complete).

### Fix
CSS: `html.sidebar-no-transition .sidebar, .sidebar-inner { transition: none !important; }`
JS: When `@media (max-width: 1235px)` changes, add `sidebar-no-transition` for 2 rAF frames (~33ms). This suppresses the initial transition, making the sidebar snap instantly to `translateX(-100%)`. Normal open/close transitions resume after the 2 frames.

### Commit
`f372e3e` — `fix: suppress sidebar flash when crossing 1235px breakpoint`
