## Sesión 48 — Fix broken dynamic right padding formula

### Prompt
Previous formula `calc(0.9vw - 342px)` was completely broken — resolved to 90px at ALL viewports because `0.9vw` is far too small to ever exceed 342px. Content was still compressing.

### Root cause
`0.9vw` at 3840px = 34.6px, which is << 342px. The clamp always hit the minimum (90px).

### Fix
Correct formula derived from linear regression through target values:
- At 1400px: right-pad = 58px
- At 1800px: right-pad = 110px
- At 3840px: right-pad = 320px

Slope = (110-58)/(1800-1400) = 0.13, Intercept = -124
→ `calc(13vw - 124px)`

### Changes
- **src/styles/global.css** (1 edit):
  - `@media (min-width: 1400px)`: `clamp(90px, calc(0.9vw - 342px), 110px)` → `clamp(58px, calc(13vw - 124px), 320px)`

### Verification
| Viewport | 13vw - 124 | Right pad | Content width |
|---|---|---|---|
| 1400px | 58px | 58px | 940px ✓ |
| 1800px | 130px → 110px | 110px | 940px ✓ |
| 2600px | 214px | 214px | 940px ✓ |
| 3840px | 375px → 320px | 320px | 940px ✓ |

## Sesión 53 — Fix page entry transition: new content now animates in

### Root cause
`display: none` → `display: block` and `classList.add('active')` happened in the same `requestAnimationFrame` callback (lines 202-205). The browser never rendered the intermediate state (`opacity: 0, translateX: 40px`), so the CSS transition on `.page` never fired.

- **src/scripts/client.js** (1 edit): Double `requestAnimationFrame` so the browser paints the initial state before adding `.active`:
```js

  requestAnimationFrame(() => {
    newPage.classList.add('active')
  })
})
```

## Sesión 64 — Fix mobile→desktop: eliminate hybrid zone + replace transitionend with setTimeout

### Prompt
When widening from mobile to desktop, contents superimpose multiple times and page needs reload. The `transitionend` approach from S63 is unreliable.

### Root cause analysis (via explore agent)
1. **Hybrid zone 1235-1280px**: `@media (max-width: 1235px)` stops applying at 1235px (sidebar snaps from `position:fixed` to `position:relative`), but `mqlExit` doesn't fire until 1280px. During these 45px, sidebar is in desktop mode while MobileProfile is still visible.
2. **`sidebar-locked` discontinuity**: When removed, `--sidebar-fade` jumps from 0 to ~0.54, causing sidebar to jump from 0px to ~151px.
3. **`is-resizing` coverage gap**: Only disables transitions on `.sidebar` and `.sidebar-overlay`, NOT on `.mobile-profile`, `.lang-switcher-floating`, or `.sidebar-inner`.
4. **`transitionend` fragility**: If opacity transition doesn't fire (element already hidden, etc.), sidebar stays locked forever.

### Changes
- **JS:** Changed `mqlExit` from `1280px` to `1235px` (synced with `@media` query). Replaced `transitionend` listener with `setTimeout(250)` for sidebar unlock. Timer is cancelable on reverse resize.
- **CSS:** Added `html.sidebar-locked .sidebar, .sidebar-inner, .lang-switcher-floating { transition: none; }` to prevent animations during lock. Changed `max-height` delay from `0.2s` to `0.25s` to sync with JS timer.
