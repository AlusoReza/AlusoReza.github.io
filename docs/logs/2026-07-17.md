## Sesión 48 — Fix broken dynamic right padding formula

### Prompt
Previous formula `calc(0.9vw - 342px)` was completely broken — resolved to 90px at ALL viewports because `0.9vw` is far too small to ever exceed 342px. Content was still compressing.

### Root cause
`0.9vw` at 3840px = 34.6px, which is << 342px. The clamp always hit the minimum (90px).

### Fix
Correct formula derived from linear regression through target values:
- At 1400px: right-pad = 58px
- At 1800px: right-pad = 110px
- At 3840px: right-pad = 320px

Slope = (110-58)/(1800-1400) = 0.13, Intercept = -124
→ `calc(13vw - 124px)`

### Changes
- **src/styles/global.css** (1 edit):
  - `@media (min-width: 1400px)`: `clamp(90px, calc(0.9vw - 342px), 110px)` → `clamp(58px, calc(13vw - 124px), 320px)`

### Verification
| Viewport | 13vw - 124 | Right pad | Content width |
|---|---|---|---|
| 1400px | 58px | 58px | 940px ✓ |
| 1800px | 130px → 110px | 110px | 940px ✓ |
| 2600px | 214px | 214px | 940px ✓ |
| 3840px | 375px → 320px | 320px | 940px ✓ |

## Sesión 49 — CSS custom properties for content padding + fix layout morphology bug

### Prompt
User can no longer change the content morphology by modifying padding in CSS. A few commits ago it worked. Wants to control content padding to expand/contract it on screen.

### Root cause
Two issues:
1. `min-width: 940px` on `.content` (added in Sesión 46) forced content to stay ≥940px regardless of padding changes
2. `max-width: 115%` on `.page` allowed page content to overflow beyond the padding area, making padding changes invisible

### Plan
1. Add CSS custom properties for content padding (`--content-pad-*`) and container width (`--app-max-width`) in `:root`
2. Replace hardcoded padding/width values with variables
3. Remove `min-width: 940px` from 1400px+ breakpoint
4. Remove `.content` padding overrides from 1400px+, 1800px+, 2600px+ breakpoints (variables handle everything)
5. Change `.page` and `.page.exiting` `max-width: 115%` → `100%`
6. Mobile breakpoints override the CSS variables instead of hardcoded padding
7. Remove unused `--content-max-width` variable

### Changes
- **src/styles/global.css** (+38/-15 lines):
  - `:root`: Added `--content-pad-top`, `--content-pad-right`, `--content-pad-bottom`, `--content-pad-left`, `--app-max-width`
  - `.app-layout`: `max-width: 1250px` → `max-width: var(--app-max-width)`
  - `.content`: padding uses `var(--content-pad-*)` variables
  - `.page` + `.page.exiting`: `max-width: 115%` → `max-width: 100%`
  - `@media (min-width: 1400px)`: removed `.content` block with `min-width: 940px` and padding override
  - `@media (max-width: 1235px)`: `.content` overrides CSS variables instead of hardcoded padding
  - `@media (max-width: 480px)`: `.content` overrides CSS variables instead of hardcoded padding
  - Removed unused `--content-max-width: 800px`
