## Session 113: Mobile profile animation fix on large→small resize + timer safety

### Prompt

The user hypothesized the issue was with vertical-only mobile profile transitions.

### Plan
1. **Root cause analysis:** The `animateMobileProfile()` function measures `scrollHeight` once at the start and animates to that fixed value. During the 350ms animation, if the viewport continues resizing, the content height changes. After the animation ends and inline height is cleared, the content pops to its new `height: auto`. Additionally, the `snapSidebarFade()` untacked setTimeout at L878 could re-trigger `animateMobileProfile(true)` on an already-visible profile, causing flicker.
2. **Micro-transition adjustment:** After the main 350ms animation, re-measure `scrollHeight`. If it differs >2px from the animated height, perform a smooth 150ms transition to the new height before cleanup.
3. **Timer safety fixes:**
   - Store `snapSidebarFade`'s `animateMobileProfile(true)` timeout in `snapProfileTimer` (was fire-and-forget)
   - Clear all timers (`mobileProfileTimer`, `snapProfileTimer`, `adjustTimer`) in `updateMobileProfile()`, `animateMobileProfile()`, and resize handler instant-hide
4. **CSS unchanged:** `height: 0` on `.mobile-profile` kept (required for PC layout stability per Session 109)

## Session 114: Fix dead zone sidebar stuck in PC layout (1236-1240px)

### Prompt
User confirmed the sidebar bug occurs between 1236-1240px (not at 1235px). The sidebar is rendered for PC layout but hidden. Bug is intermittent. Root cause: `clearTimeout(resizeTimer)` at L578 in `handleMobileProfile()` cancels the 1000ms timer that would call `snapSidebarFade()`.

### Root cause analysis
When crossing from <1235px to 1236-1240px:
1. `mqlBreakpoint` change fires → `handleMobileProfile()` adds `sidebar-delayed`, starts 350ms timer
2. `resize` handler fires → adds `is-resizing`, starts 1000ms timer → `snapSidebarFade()`
3. At 350ms: `handleMobileProfile()` removes `sidebar-delayed`, removes `is-resizing`, **`clearTimeout(resizeTimer)`** kills the 1000ms timer
4. `snapSidebarFade()` never runs → `sidebar-midpoint-mode` never added → sidebar stays in PC layout with `--sidebar-fade ≈ 0`

Intermittent because `mouseup` handler also calls `snapSidebarFade()` if `is-resizing` is active, but `handleMobileProfile()` clears `is-resizing` at L577 before `mouseup` fires (if user releases mouse after 350ms).

### Plan
Replace the manual sidebar transition block in `handleMobileProfile()` (L575-592) with a single `snapSidebarFade()` call. This:
- Eliminates `clearTimeout(resizeTimer)` → resize handler's 1000ms fallback still works
- Eliminates manual sidebar flex/width transition → `snapSidebarFade()` handles state correctly
- Ensures `sidebar-midpoint-mode` is always added when entering 1236-1285px range

### Changes
- `src/scripts/client.js` L575-592: Replaced 17-line block (`clearTimeout(resizeTimer)` + manual sidebar flex/width transition) with 3-line block: remove `sidebar-delayed`, remove `is-resizing`, call `snapSidebarFade()`
