## Session 117: Fix mobile profile animation order — prevent flash on rapid resize

### Prompt
When rapidly resizing from a large window (>1235px) to a small one (<1235px), the top half of the mobile profile appears suddenly without animation first, then the bottom half animates in. Root cause: in `animateMobileProfile(show=true)`, `style.height=''` (line 526) before `classList.add('--visible')` (line 527) causes CSS `height:auto` to take effect momentarily. The forced layout during `scrollHeight` measurement can allow a composite frame at full height.

### Plan
1. Invert operation order in `animateMobileProfile(show=true)`: set `style.height='0'` BEFORE adding `--visible` class, ensuring element is never at full height before animation starts.
2. Add scrollHeight fallback: if `scrollHeight` returns 0 (possible with `height:0` + `overflow:clip` + `grid-template-rows:1fr`), measure at `height:auto` as fallback.
3. Verify no regressions in all other code paths (updateMobileProfile, animateMobileProfile(false), handleMobileProfile, snapSidebarFade, resize handler, ResizeObserver).

### Changes
- `src/scripts/client.js` L525-537: Inverted operation order in `animateMobileProfile(show=true)`:
  - Before: `style.height=''` → `classList.add('--visible')` → `scrollHeight` → `style.height='0'`
  - After: `style.height='0'` → `offsetHeight` → `classList.add('--visible')` → `scrollHeight` → fallback if `h <= 0` → `offsetHeight`
  - Added fallback: if `scrollHeight` returns 0 (grid `1fr` at 0 height), temporarily set `height:auto` to measure real content height.

## Session 124: Fix sidebar animation during resize — @keyframes entrance

### Prompt
The sidebar doesn't animate during resize — it only animates when you stop. The user discovered this happens at ANY speed, not just intermediate. Root cause: `is-resizing` adds `transition: none !important` on `.sidebar`, so when `sidebar-delayed` is removed by the 350ms timer, the sidebar can't transition from 0→visible. The next resize event (~16ms later) re-adds `is-resizing`, cutting off any transition that started.

### Plan
Replace CSS transition approach with `@keyframes sidebar-entrance` animation. CSS animations are NOT suppressed by `transition: none` — they're a separate mechanism. Capture `--sidebar-fade` value at animation start as `--entrance-target` (static custom property). On `animationend`, cleanup + `snapSidebarFade()`.

### Changes
- `src/styles/global.css` L84-96: Added `@keyframes sidebar-entrance` (opacity/flex/width 0→target) and `.sidebar.sidebar-entrance` class with `animation: sidebar-entrance 0.3s ease forwards`.
- `src/scripts/client.js` L580-595: Replaced inline-removal fix with animation logic: capture `--sidebar-fade` via `getComputedStyle`, set `--entrance-target` on sidebar, add `sidebar-entrance` class, `animationend` listener to cleanup + `snapSidebarFade()`.
